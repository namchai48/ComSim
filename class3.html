2 Discrete Event Simulation — เวลา "กระโดด" ทีละเหตุการณ์

คอมพิวเตอร์ไม่ได้เดินเวลาไปทีละวินาที — มันข้ามไปยังเหตุการณ์ (event) ถัดไปทันที เพราะระหว่างเหตุการณ์ไม่มีอะไรเปลี่ยนแปลง

🧩 ศัพท์หลักของ DES

คำศัพท์ความหมาย (ตัวอย่างธนาคาร)
Entityสิ่งที่ไหลผ่านระบบ → ลูกค้า
Eventเหตุการณ์ที่เปลี่ยนสถานะระบบ → มาถึง / เริ่มบริการ / ออกจากระบบ
Server / Resourceทรัพยากรจำกัดที่ให้บริการ → teller
Queueคิวที่เกิดเมื่อทรัพยากรไม่ว่าง
Simulation clockนาฬิกาของโมเดล — กระโดดจาก event หนึ่งไป event ถัดไป
Stateสถานะระบบ เช่น จำนวนคนในคิว, teller ว่าง/ไม่ว่าง

⏭️ ทำไมต้อง "กระโดด"?

ถ้าเดินเวลาทีละ 1 วินาที ธนาคาร 8 ชม. = 28,800 รอบคำนวณ ทั้งที่ส่วนใหญ่ "ไม่มีอะไรเกิดขึ้น" — DES จึงเก็บรายการเหตุการณ์อนาคต (future event list) แล้วกระโดดไปยังเหตุการณ์ที่ใกล้ที่สุด อัปเดตสถานะ แล้วกระโดดต่อ

เชื่อมกับครั้งที่ 1 Monte Carlo ใน Excel สุ่มผลลัพธ์ "ทั้งวัน" ในครั้งเดียว — แต่ DES จำลอง พลวัตรายเหตุการณ์ จึงตอบคำถามเรื่อง "คิว" และ "เวลารอ" ได้ ซึ่ง Monte Carlo แบบตารางทำไม่ได้

🕹️ ลองเล่น: เดินเหตุการณ์เองทีละ event (ธนาคาร 1 teller)

ลูกค้า 5 คน มาถึงที่ t = 0, 2, 3, 7, 8 นาที · เวลาบริการ 4, 3, 2, 3, 2 นาที — กด "event ถัดไป" แล้วสังเกตว่านาฬิกา ไม่เดินทีละนาที

SIMULATION CLOCK (นาที)—
กดปุ่ม "Event ถัดไป" เพื่อเริ่ม...
สังเกต นาฬิกากระโดด 0 → 2 → 3 → 4 → 7 → 8 → 9 → 12 → 14 — มีแค่ 10 เหตุการณ์ ใน 9 จุดเวลา (ที่ t=7 มีทั้งลูกค้าเสร็จและมาถึงพร้อมกัน) แทนที่จะคำนวณ 14 นาที × 60 วินาที และนี่คือสิ่งที่ SimPy ทำให้เราอัตโนมัติ

3) loop while

i = 0

    i += 1
# ใน SimPy: while True = สร้างลูกค้าเรื่อย ๆ

4) generator & yield ⭐ สำคัญสุด

def customer():
    yield "รอ teller ว่าง"
    yield "ใช้เวลาบริการ 4 นาที"

yield = "หยุดตรงนี้ไว้ก่อน แล้วค่อยกลับมาทำต่อ" — SimPy ใช้กลไกนี้ พักกระบวนการ ระหว่างรอคิวหรือรอเวลาบริการ

5) with — ขอใช้ทรัพยากร แล้ว "คืนอัตโนมัติ"

with bank.request() as req:   # ขอใช้ teller 1 ตัว
    yield req                    # รอจนได้
    ...                          # รับบริการอยู่ในบล็อกนี้
# ⬅ พ้นบล็อก with = คืน teller ให้คนถัดไปทันที ไม่ต้องเขียนโค้ดคืนเอง

with ... as ...: คือการ "ยืมของแล้วระบบคืนให้เองเมื่อจบบล็อก" — ถ้าลืมคืน teller ลูกค้าคนถัดไปจะรอตลอดกาล บล็อก with จึงกันบั๊กสำคัญนี้ให้เราอัตโนมัติ (จะเจอบรรทัดนี้ในโค้ดจริงหัวข้อ 5)

เซลล์ที่ 2 — list + while + function + ค่าสุ่ม ทบทวนข้อ 1–3

import random, statistics
random.seed(42)                       # ล็อกค่าสุ่มให้ทุกคนได้ผลเหมือนกัน

wait_times = []                       # list ว่าง
i = 0
while i < 5:                          # สุ่มเวลารอสมมติ 5 คน
    wait_times.append(random.expovariate(1/4.0))
    i += 1

def summary(data):                    # ฟังก์ชันสรุปค่าเฉลี่ย
    return round(statistics.mean(data), 2)

print("เวลารอ 5 คน:", [round(w,2) for w in wait_times])
print("เฉลี่ย:", summary(wait_times), "นาที")
ผลลัพธ์ที่ควรได้ (seed 42 — ต้องตรงเป๊ะทุกเครื่อง)เวลารอ 5 คน: [4.08, 0.1, 1.29, 1.01, 5.33] เฉลี่ย: 2.36 นาที
สังเกตได้เลข ชุดเดียวกัน กับเพื่อนทั้งห้องเพราะ random.seed(42) — ถ้าลบบรรทัดนี้ออกแล้วรันใหม่ ตัวเลขจะเปลี่ยนทุกครั้ง นี่คือเหตุผลที่ Lab หัวข้อ 6 บังคับใช้ seed 42 เพื่อให้เทียบผลกันได้

!pip install simpy -q                 # ⬅ เครื่องหมาย ! = สั่งติดตั้งโปรแกรม ไม่ใช่โค้ด Python
import simpy
print("SimPy version:", simpy.__version__)

def customer(env, name, arrive_at):
    yield env.timeout(arrive_at)      # รอจนถึงเวลาที่ลูกค้ามาถึง
    print(f"{env.now:5.1f} นาที : {name} มาถึงธนาคาร")
    yield env.timeout(4)              # ใช้บริการ 4 นาที
    print(f"{env.now:5.1f} นาที : {name} ออกจากธนาคาร")

env = simpy.Environment()
env.process(customer(env, "ลูกค้า A", 0))
env.process(customer(env, "ลูกค้า B", 3))

SimPy คือไลบรารี Discrete Event Simulation ของ Python — โครงสร้างมีแค่ 4 อย่างที่ต้องรู้

องค์ประกอบ SimPyทำหน้าที่ในโมเดลธนาคาร
simpy.Environment()ตัวจัดการ เวลาและเหตุการณ์ ทั้งหมด (simulation clock + future event list)นาฬิกาของธนาคารจำลอง — อ่านเวลาปัจจุบันได้จาก env.now
env.process(...)ลงทะเบียน generator function (ฟังก์ชันที่มี yield) ให้เป็นกระบวนการในโมเดลลูกค้าแต่ละคน / ตัวปั๊มลูกค้าเข้าระบบ
simpy.Resource(env, capacity=c)ทรัพยากรจำกัด — ถ้าเต็ม ผู้ขอจะต่อคิวอัตโนมัติ (FIFO)teller 3 คน (capacity=3)
yield env.timeout(t)ให้กระบวนการ ใช้เวลา t หน่วย ในโมเดล (ไม่ใช่เวลาจริงของเครื่อง)เวลาให้บริการ ~4 นาที / เวลารอลูกค้าคนถัดไปมาถึง

เห็นทั้ง 4 ชิ้นทำงานพร้อมกันในโค้ด 10 บรรทัด

นี่คือโค้ดตัวเดียวกับ แบบฝึกเซลล์ที่ 3 ที่ทดลองใน Colab ไปแล้วในหัวข้อ 3 — คราวนี้มาดูว่าแต่ละบรรทัดคือองค์ประกอบไหน

def customer(env, name, arrive_at):
    yield env.timeout(arrive_at)      # รอจนถึงเวลาที่ลูกค้ามาถึง
    print(f"{env.now:5.1f} นาที : {name} มาถึงธนาคาร")
    yield env.timeout(4)              # ใช้บริการ 4 นาที
    print(f"{env.now:5.1f} นาที : {name} ออกจากธนาคาร")

env = simpy.Environment()
env.process(customer(env, "ลูกค้า A", 0))
env.process(customer(env, "ลูกค้า B", 3))
env.run()

env = simpy.Environment()① Environment — สร้าง "โลกจำลอง" 1 ใบ พร้อมนาฬิกาที่เริ่มนับจาก 0 และปฏิทินเหตุการณ์ที่ยังว่างเปล่า
env.process(customer(env, "ลูกค้า A", 0))② process — ส่งลูกค้า A เข้าไปอยู่ในโลกนี้ · ยังไม่มีอะไรเกิดขึ้น แค่จองคิวไว้ในปฏิทิน · เรียกซ้ำอีกครั้งก็ได้ลูกค้า B (คนละกระบวนการ ต่างคนต่างเดินเรื่องของตัวเอง)
yield env.timeout(4)③ timeout + yield — "หยุดตรงนี้ 4 หน่วยเวลา" · yield คือจุดที่กระบวนการพักไว้ แล้วคืนคิวให้ Environment ไปทำคนอื่นต่อ
env.run()เริ่มเดินเครื่อง — Environment หยิบเหตุการณ์ที่เวลาน้อยสุดออกมา เลื่อนนาฬิกาไปที่เวลานั้น ปลุกกระบวนการที่รออยู่ วนแบบนี้จนปฏิทินหมด (หรือถึงเวลา until=)
env.nowอ่านเวลาปัจจุบันของโลกจำลอง — ในตัวอย่างนี้คือเลข 0.0 / 3.0 / 4.0 / 7.0 ที่พิมพ์ออกมา
ยังขาดชิ้นที่ 4: Resource ตัวอย่างนี้ยังไม่มีเคาน์เตอร์จำกัด — ลูกค้า A และ B จึงรับบริการพร้อมกันได้โดยไม่ต้องรอกันเลย (เหมือนธนาคารมีเคาน์เตอร์ไม่จำกัด) · พอเพิ่ม simpy.Resource(env, capacity=3) เข้าไป ระบบคิวจริง ๆ ถึงจะเกิด — ไปดูกันในหัวข้อ 5

ก่อนอ่านโค้ด — เล่าเป็นภาษาคนก่อน 1 รอบ

ลองนึกภาพตัวเองเป็น ลูกค้า 1 คน ที่เดินเข้าธนาคาร สิ่งที่คุณทำมีแค่ 5 จังหวะ — โค้ดข้างล่างก็เขียนตาม 5 จังหวะนี้เป๊ะ ๆ ไม่มีอะไรมากกว่านั้น

ชีวิตของลูกค้า 1 คน (pseudocode ภาษาไทย)
  1. เดินเข้าธนาคาร → ดูนาฬิกาแล้วจำไว้ว่ามาถึงกี่โมง
  2. หยิบบัตรคิว (ขอใช้เคาน์เตอร์ 1 ช่อง)
  3. ยืนรอ จนกว่าจะมีเคาน์เตอร์ว่าง — ถ้าว่างอยู่แล้วก็เดินเข้าได้เลย ไม่ต้องรอ
  4. พอถึงคิว → จดว่ารอไปกี่นาที (เวลาตอนนี้ − เวลาที่มาถึง)
  5. รับบริการ ใช้เวลาสุ่ม ๆ ราว 4 นาที แล้ว ลุกออก คืนเคาน์เตอร์ ให้คนถัดไป
อุปมาที่ต้องจำให้ได้แค่ 3 อย่าง 🎫 bank.request() = หยิบบัตรคิว  ·  ⏸️ yield = "หยุดตรงนี้ รอก่อน"  ·  🚪 with จบบล็อก = ลุกออกจากเคาน์เตอร์ คืนที่ให้คนอื่นอัตโนมัติ

import simpy, random, statistics

NUM_TELLERS  = 3      # จำนวน teller (Resource capacity)
INTERARRIVAL = 2.0    # นาที — เฉลี่ย 30 คน/ชม.
SERVICE_MEAN = 4.0    # เวลาบริการเฉลี่ย (นาที)
SIM_TIME     = 480    # 8 ชั่วโมง = 480 นาที
wait_times = []       # เก็บเวลารอของลูกค้าทุกคน

def customer(env, name, bank):
    arrive = env.now                      # จดเวลามาถึง
    with bank.request() as req:        # ขอใช้ teller
        yield req                        # รอจนได้ teller (ต่อคิวอัตโนมัติ)
        wait_times.append(env.now - arrive)  # เวลารอ = ตอนได้ - ตอนมา
        yield env.timeout(random.expovariate(1.0/SERVICE_MEAN))  # รับบริการ

def setup(env):
    bank = simpy.Resource(env, capacity=NUM_TELLERS)
    i = 0
    while True:                          # ปั๊มลูกค้าไปเรื่อย ๆ
        yield env.timeout(random.expovariate(1.0/INTERARRIVAL))
        i += 1
        env.process(customer(env, i, bank))  # ลูกค้าคนที่ i เข้าระบบ

random.seed(42)                # ล็อกค่าสุ่ม → ผลซ้ำได้ (reproducible)
env = simpy.Environment()      # สร้างนาฬิกา + ตัวจัดการเหตุการณ์
env.process(setup(env))
env.run(until=SIM_TIME)        # รันจนถึงนาทีที่ 480

print("รับบริการ:", len(wait_times), "คน")
print("เวลารอเฉลี่ย:", round(statistics.mean(wait_times), 2), "นาที")
print("เวลารอสูงสุด:", round(max(wait_times), 2), "นาที")
# ผลที่ทดสอบแล้ว (seed 42): ลูกค้า ~225 คน, รอเฉลี่ย ~0.7 นาที

        wait_times.append(env.now - arrive)
        st = random.expovariate(1.0/SERVICE_MEAN)
        busy_time += min(st, SIM_TIME - env.now)  # ⬅ เพิ่ม (นับเฉพาะช่วงก่อนธนาคารปิด)
        yield env.timeout(st)
    system_times.append(env.now - arrive)   # ⬅ เพิ่ม

# ---- หลัง env.run(...) ----
print("เวลาในระบบเฉลี่ย:", round(statistics.mean(system_times),2), "นาที")
print("utilization ≈", round(busy_time/(NUM_TELLERS*SIM_TIME),3))
print("คนในคิวเฉลี่ย Lq ≈", round(sum(wait_times)/SIM_TIME,2))  # Little's law
ทำไม Lq = ผลรวมเวลารอ ÷ เวลาทั้งหมด? นี่คือ Little's Law ในรูปที่ใช้งานจริง — ลูกค้าที่รอ 3 นาทีจะ "นับเป็นคนในคิว" อยู่ 3 นาที ดังนั้นผลรวมเวลารอของทุกคน = จำนวนคน-นาทีที่คิวสะสมไว้ทั้งวัน หารด้วย 480 นาที = จำนวนคนในคิว เฉลี่ย ณ ขณะใดขณะหนึ่ง (สูตรทฤษฎีคือ Lq = λ × Wq — ลองแทนค่าเช็คดูได้ว่าตรงกัน)
busy_time ต้องตัดที่เวลาปิด ลูกค้าที่เริ่มรับบริการใกล้นาทีที่ 480 อาจใช้เวลาบริการ "เลย" เวลาปิดไป — ถ้าบวก st เต็ม ๆ utilization จะเกินจริง (บางกรณีเกิน 100%!) จึงใช้ min(st, SIM_TIME - env.now) นับเฉพาะส่วนที่อยู่ในช่วงจำลอง

📚 ตารางแปลง: ผล fit จากครั้งที่ 2 → โค้ด Python

การบ้านครั้งนี้ต้องเปลี่ยนโมเดลธนาคารให้เป็นระบบของกลุ่มคุณ — จุดที่ต้องแก้คือค่าที่ใส่ใน env.timeout(...) ให้ตรงกับการแจกแจงที่ fit ได้จริง · ใช้ตารางนี้คู่กับผล Chi-square ของกลุ่ม (การแจกแจงหลัก 5 ตัวเดียวกับครั้งที่ 2)

การแจกแจงใช้กับอะไรพารามิเตอร์ที่ fit ได้Excel (ครั้งที่ 2)Python
Uniform รู้แค่ช่วงต่ำสุด–สูงสุด ไม่มีข้อมูลอื่น (ประมาณคร่าว ๆ) a = min, b = max
mean = (a+b)/2		 =a+(b-a)*RAND() random.uniform(a, b)
Triangular ข้อมูลน้อย ใช้ความเห็นผู้เชี่ยวชาญ: min / mode / max a = min, c = mode, b = max
mean = (a+c+b)/3		 ต้องเขียนสูตรเงื่อนไขยาว random.triangular(a, b, c)
⚠️ ลำดับคือ low, high, mode — mode อยู่ท้าย ไม่ใช่ตรงกลาง
Normal ค่าลอยรอบค่าเฉลี่ยแบบสมมาตร เช่น น้ำหนัก/ขนาดสินค้า ยอดรวมต่อวัน mean (μ), SD (σ) =NORM.INV(RAND(), mean, sd) random.normalvariate(mean, sd)
⚠️ ให้ค่าติดลบได้ — ไม่เหมาะกับ "เวลา" ถ้าจำเป็นต้องใช้ ให้ครอบด้วย max()
Exponential เวลาระหว่างเหตุการณ์ที่เกิดแบบสุ่มไร้ความจำ — โดยเฉพาะ เวลาระหว่างลูกค้ามาถึง mean อย่างเดียว (λ = 1/mean) =-mean*LN(RAND()) random.expovariate(1.0/mean)
⚠️ ต้องใส่ 1/mean ไม่ใช่ mean
Poisson จำนวนนับต่อช่วงเวลา เช่น ลูกค้ากี่คน/ชม. ออเดอร์กี่รายการ/วัน (ไม่ใช่ค่าเวลา) λ = จำนวนเฉลี่ยต่อช่วง ไม่มีฟังก์ชันผกผันโดยตรง — ใช้ตาราง lookup สะสมความน่าจะเป็น หรือประมาณด้วย =BINOM.INV(n, λ/n, RAND()) (n ใหญ่ ๆ เช่น 1000) numpy.random.poisson(lam)
โมดูล random ไม่มีให้ ต้อง import numpy as np
Poisson กับ Exponential คือเรื่องเดียวกัน — คนละมุมมอง ถ้าจำนวนลูกค้าต่อชั่วโมงเป็น Poisson(30) แล้ว เวลาระหว่างลูกค้าสองคนจะเป็น Exponential ที่มี mean = 60/30 = 2 นาที โดยอัตโนมัติ · ใน SimPy เราสร้างลูกค้าทีละคน จึงใช้ Exponential เสมอ — อย่าเผลอเอา Poisson ไปใส่ env.timeout() (Poisson ใช้ตอนสรุปผลหรือกำหนดจำนวนงานต่อรอบเท่านั้น)
