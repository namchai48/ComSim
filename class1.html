1 Simulation คืออะไร?

การจำลองสถานการณ์บนคอมพิวเตอร์ — ทดลองกับ "แบบจำลอง" ของระบบจริง โดยไม่ต้องเสี่ยงกับของจริง

Computer simulation — experimenting with a model of a real system, without touching the real thing.

นิยาม / Definition

“Simulation คือการทดลองกับแบบจำลองของระบบจริงบนคอมพิวเตอร์ โดยปล่อยให้เวลาเดินไป เพื่อดูว่าระบบจะทำงานอย่างไร” — เหมือนสร้างร้านกาแฟจำลองในคอมพิวเตอร์ แล้วลองเปิดร้านเป็นพันวันโดยไม่ต้องเปิดร้านจริง (Robinson, 2014)

“Simulation is experimenting with a model of a real system on a computer, letting time advance to see how the system behaves” — like building a virtual coffee shop and running it for a thousand days without opening a real one. (Robinson, 2014)

💡 เปรียบเทียบ / Analogy เหมือนเกม The Sims หรือเกมบริหารเมือง — เราตั้งกฎ ปล่อยระบบเดิน แล้วสังเกตผล ต่างกันตรงที่ของเราใช้ ข้อมูลจริง และเอาผลไป ตัดสินใจธุรกิจ Like The Sims or a city-builder game — set the rules, let it run, observe. The difference: we use real data and turn results into business decisions.
⚠️ เข้าใจผิดบ่อย / Common myth นักศึกษามักคิดว่า simulation = แอนิเมชัน 3 มิติสวยๆ — จริงๆ แอนิเมชันเป็นแค่ "เปลือก" หัวใจคือ ตรรกะและสถิติ ข้างใน Students often think simulation = pretty 3D animation. The animation is just the shell — the heart is the logic and statistics inside.

ใช้ทำอะไรในงานโลจิสติกส์? / Uses in logistics

🏭 ออกแบบคลังสินค้า — กี่ประตูรับ-ส่ง? ทางเดินกว้างเท่าไร? พนักงานกี่คน?🏭 Warehouse design — how many dock doors, aisle width, staff?
🚚 วิเคราะห์การไหลของสินค้าและหา คอขวด (bottleneck)🚚 Analyze freight flow and find bottlenecks.
📊 ดู อัตราการใช้งาน (utilization) ของรถยก/เครื่องจักร และสินค้าคงคลัง📊 Measure utilization of forklifts/machines and inventory levels.
🤖 ทดสอบระบบ AS/RS, หุ่นยนต์, IoT ยุค Industry 4.0 ก่อนลงทุนจริง🤖 Test AS/RS, robots, IoT (Industry 4.0) before real investment.

2 ทำไมต้อง Simulate & เมื่อไร "ไม่ควร"

Simulation ให้ ข้อมูล ไม่ใช่ คำตอบสำเร็จรูป — มันบอกว่า "ถ้าทำแบบนี้จะเกิดอะไร" แต่คนยังต้องตัดสินใจ

Simulation gives you insight, not answers — it tells you "what happens if", but humans still decide.

✅ ควรใช้เมื่อ / Use when

  • อยากทำนายผล ก่อนสร้างจริง — ลดความเสี่ยง
  • Predict results before building — reduce risk.
  • ทดลองนโยบายใหม่โดยไม่กระทบระบบจริง
  • Test new policies without disrupting the real system.
  • หาคอขวด / ดูพฤติกรรมเมื่อเจอเหตุไม่ปกติ
  • Find bottlenecks / see behavior under abnormal events.
  • ระบบมี ความสุ่ม และ ความสัมพันธ์ซับซ้อน
  • System has randomness and complex interactions.

🚫 ไม่ควรใช้เมื่อ / Avoid when

  • แก้ได้ด้วย สูตร/สามัญสำนึก อยู่แล้ว (ใช้ค้อนทุบมด)
  • A formula or common sense already solves it.
  • ทดลองกับระบบจริงง่ายและถูกกว่า
  • Testing the real system is cheaper & easier.
  • ไม่มีข้อมูลเลยแม้แต่ค่าประมาณ
  • No data at all, not even estimates.
  • เวลา/งบ/คนไม่พอจะพิสูจน์ว่าแบบจำลองถูกต้อง
  • Not enough time/budget/people to validate the model.
❓ ชวนคิด / Think “อยากรู้ว่าเหรียญออกหัวกี่ % — ควรใช้ simulation ไหม? เพราะอะไร?” “Want to know the % a coin lands heads — should you use simulation? Why?”

3 Discrete vs Continuous

วิชานี้เน้น Discrete Event — ระบบเปลี่ยนสถานะเป็น "เหตุการณ์" ทีละจุดเวลา (ลูกค้ามาถึง / เริ่มบริการ / ออก)

This course focuses on Discrete Event systems — state changes at discrete events (arrive / start service / leave).

Discrete — เหมือน นาฬิกาดิจิทัล ที่กระโดดเป็นขั้น ค่าคงที่ระหว่างเหตุการณ์ (เส้นขั้นบันไดสีเขียว)Discrete — like a digital clock jumping in steps; constant between events (green staircase).
Continuous — เหมือน เข็มนาฬิกา ที่ไหลลื่น เปลี่ยนตลอดเวลา เช่น ระดับน้ำ อุณหภูมิ (เส้นโค้งสีเหลือง)Continuous — like an analog hand flowing smoothly; always changing, e.g. water level, temperature (amber curve).
⚠️ กับดัก / Trap “เวลาเดินต่อเนื่อง = continuous” — ผิด! เรามองที่ สถานะระบบ ไม่ใช่ตัวเวลา แม้เวลาจะไหลต่อเนื่อง แต่ถ้าสถานะเปลี่ยนเป็นจุดๆ ก็คือ discrete “Time flows continuously, so it's continuous” — Wrong! We look at the system state, not the clock. Even with continuous time, if the state jumps at points, it's discrete.

แล้ว DES กับ DESS คืออะไร? / What are DES & DESS?

สองคำนี้เจอบ่อยมากในวิชานี้ (ชื่อตำราหลักก็คือ Discrete-Event System Simulation) — ต่างกันนิดเดียวแต่สำคัญ

You'll see these two terms a lot (the main textbook is literally Discrete-Event System Simulation) — a small but important difference.

DES · Discrete Event Simulation “เทคนิค/วิธีรัน” — คอมพิวเตอร์ กระโดดจากเหตุการณ์หนึ่งไปเหตุการณ์ถัดไป (event-to-event) ไม่เดินเวลาทีละวินาทีเท่าๆ กัน เพราะระหว่างสองเหตุการณ์สถานะไม่เปลี่ยน จึงข้ามได้เลย → รันเร็ว

🏦 ตัวอย่าง: โมเดลธนาคาร นาฬิกาจำลองกระโดดไปที่เวลา “ลูกค้ามาถึง → เริ่มบริการ → บริการเสร็จ” เท่านั้น ไม่สนใจช่วงว่างระหว่างนั้น “The technique / engine” — the computer jumps from one event to the next (event-to-event) instead of ticking every second. Between two events the state doesn't change, so it skips ahead → fast.

🏦 Example: a bank model — the simulation clock only jumps to “customer arrives → service starts → service ends,” ignoring the idle gaps in between.
DESS · Discrete Event System Simulation “ระบบ + การจำลอง” — การจำลอง ระบบที่ตัวแปรสถานะเปลี่ยนเฉพาะ ณ จุดเวลาที่นับได้ (เมื่อเกิดเหตุการณ์) เป็นกรอบ/หัวข้อของทั้งวิชา ครอบคลุมทั้งการสร้างแบบจำลองและการวิเคราะห์

📦 ตัวอย่าง: จำลองทั้ง “ระบบคลังสินค้า” — สินค้าเข้า/จัดเก็บ/หยิบ/ส่งออก แต่ละอย่างเป็นเหตุการณ์ที่เปลี่ยนสถานะ (จำนวนสินค้า, คิวงาน, รถยกว่าง/ไม่ว่าง) “The system + its simulation” — simulating a system whose state variables change only at countable points in time (when events occur). It's the framing of the whole course, covering both modeling and analysis.

📦 Example: modeling an entire “warehouse system” — receive/store/pick/ship, each an event that changes state (stock count, job queue, forklift free/busy).
พูดง่ายๆ: DESS = ระบบที่เราศึกษา (สถานะเปลี่ยนเป็นจุดๆ) · DES = เครื่องยนต์ที่ใช้ขับ (รันแบบกระโดดตามเหตุการณ์) In short: DESS = the system we study (state changes at discrete points) · DES = the engine that runs it (jumps event to event).

ตัวอย่างเทียบกัน / Side-by-side examples

🟢 Discrete Event (เน้นในวิชานี้) คิวธนาคาร/ATM · สายการผลิต · คลังสินค้า (รับ-เก็บ-หยิบ-ส่ง) · ท่าเรือ/ตู้คอนเทนเนอร์ · เช็กอินสนามบิน · call center · ห้องฉุกเฉินโรงพยาบาล Bank/ATM queues · production lines · warehouses (receive-store-pick-ship) · ports/containers · airport check-in · call centers · hospital ER
🟡 Continuous (ไม่ใช่โฟกัสหลัก) ระดับน้ำในถัง · อุณหภูมิเตาเผา · ความเร็ว/ตำแหน่งของรถ · การไหลของของเหลวในท่อ · ปฏิกิริยาเคมี · การเติบโตของประชากรแบบต่อเนื่อง Water tank level · furnace temperature · vehicle speed/position · fluid flow in pipes · chemical reactions · continuous population growth
💡 เชื่อมกับภาพเคลื่อนไหวด้านบน / Link to the animation เส้นขั้นบันไดสีเขียว = DESS: สถานะกระโดดเป็นขั้นเฉพาะตอนเกิดเหตุการณ์ (จุดสีน้ำเงิน) ส่วน DES คือกลไกที่ทำให้นาฬิกาจำลอง “กระโดด” ไปทีละจุดเหล่านั้น The green staircase = DESS: state jumps only at events (blue dots). DES is the mechanism that makes the simulation clock “jump” from one of those points to the next.

7 Lab: Monte Carlo ใน Excel

แนวคิดสำคัญที่สุดของคาบ: “ผลของ simulation คือตัวแปรสุ่ม” — ผลเปลี่ยนทุกครั้งที่สุ่มใหม่ แต่ ค่าเฉลี่ยค่อนข้างคงที่ ส่วนนี้ให้ลงมือทำใน Excel เอง เพื่อฝึกใช้งานฟังก์ชันจริง

The big idea: “a simulation's output is a random variable” — it changes each re-roll, but the average stays fairly stable. Do this hands-on in Excel to build real spreadsheet skills.

☕ B. ยอดลูกค้าร้านกาแฟ / Customers — normal

Sheet B_ยอดลูกค้า — 1,000 days (mean 120, SD 20), then a binned frequency table.
